# 4.3.9.1. 표 개체

Tag ID: HWPTAG_TABLE

**표 74: 표 개체**

| 자료형 | 길이(바이트) | 설명 |
|---|---|---|
| BYTE stream | n | 개체 공통 속성(표 68) |
| BYTE stream | n2 | 표 개체 속성(표 75) |
| BYTE stream | n3 | 셀 리스트(표 79). 셀 크기×셀 개수 |
| 전체 길이 | 가변 | n+n2+n3 |

**표 75: 표 개체 속성**

| 자료형 | 길이(바이트) | 설명 |
|---|---|---|
| UINT32 | 4 | 속성(표 76) |
| UINT16 | 2 | RowCount |
| UINT16 | 2 | nCols |
| HWPUNIT16 | 2 | CellSpacing |
| BYTE stream | 8 | 안쪽 여백(표 77) |
| BYTE stream | 2×row | Row Size |
| UINT16 | 2 | Border Fill ID |
| UINT16 | 2 | Valid Zone Info Size (5.0.1.0 이상) |
| BYTE stream | 10×zone | 영역 속성(표 78) (5.0.1.0 이상) |
| 전체 길이 | 가변 | 22+(2×row)+(10×zone) |

**표 76: 표 속성** — bit 0-1 쪽 경계 나눔, bit 2 제목 줄 자동 반복. **표 77: 안쪽 여백** — HWPUNIT16 좌/우/상/하 각 2, 전체 8. **표 78: 영역 속성** — UINT16 시작열, 시작행, 끝열, 끝행, 테두리채우기 ID. 전체 10.

**표 79: 셀 리스트** — 문단 리스트 헤더(표 65) + 셀 속성(표 80). 전체 가변 26+n.

---

## 파싱 주의사항 (Parsing Notes)

표 65에 명시된 ListHeader는 **6바이트**이지만, 실제 구현에서는 **8바이트**일 수 있다 (UINT16 + UINT16 + UINT32). 따라서 **셀 속성(표 80)을 파싱할 때는 ListHeader의 실제 크기를 확인하여 올바른 offset(6 또는 8바이트)을 사용**해야 한다.

**표 80: 셀 속성**

| 자료형 | 길이(바이트) | 설명 |
|---|---|---|
| UINT16 | 2 | 셀 주소 Column |
| UINT16 | 2 | 셀 주소 Row |
| UINT16 | 2 | 열 병합 개수 |
| UINT16 | 2 | 행 병합 개수 |
| HWPUNIT | 4 | 셀 폭 |
| HWPUNIT | 4 | 셀 높이 |
| HWPUNIT16 array[4] | 8 | 4방향 여백 |
| UINT16 | 2 | 테두리/배경 ID |
| 전체 길이 | 26 | |
