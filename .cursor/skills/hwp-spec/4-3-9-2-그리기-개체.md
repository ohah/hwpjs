# 4.3.9.2. 그리기 개체(선, 사각형, 타원, 호, 다각형, 곡선)

그리기 개체는 먼저 그리기 개체 공통 속성을 serialize한 다음 개체 요소 속성을 serialize한다.

**표 81: 그리기 개체 공통 속성** — 개체 요소 속성(표 82) + 테두리 선 정보(표 86) 11바이트 + 채우기(표 28) + 글상자 리스트 12바이트(있으면). 전체 가변 23+n+n2.

**표 82: 개체 요소 속성** — UINT32 개체 컨트롤 ID, UINT32 개체 컨트롤 ID(두 번째). 전체 8.

**참고**: 실제 HWP 파일에서는 **모든 SHAPE_COMPONENT 레코드에서 개체 컨트롤 ID가 두 번 기록**된다. pyhwp, hwp.js도 항상 두 번째 ID를 읽고 있다. **표 83: 개체 요소 속성** — INT32 그룹 내 X/Y offset, WORD 그룹 횟수, WORD local file version, UINT32 초기/현재 폭·높이, UINT32 속성(flip), HWPUNIT16 회전각, INT32 회전 중심 x/y, Rendering 정보(표 84). 전체 가변 42+n. **표 84: Rendering 정보** — WORD cnt(matrix 쌍 개수), 48바이트 translation, cnt×48×2 scale/rotation. **표 85: matrix** — double array[6] 48바이트 (3×2). **표 86: 테두리 선** — COLORREF, INT16 굵기, UINT32 속성(표 87), BYTE Outline style(표 88). 전체 11. **표 87, 88**: 선 종류·화살표·Outline style. **표 89: 글상자 텍스트** — 12바이트 속성(표 90) + 문단 리스트(표 65). **표 90**: 글상자 여백·최대 폭.

**4.3.9.2.1 개체 요소** Tag: HWPTAG_SHAPE_COMPONENT. **4.3.9.2.2 선** Tag: HWPTAG_SHAPE_COMPONENT_LINE. 표 91/92: 개체 공통 + 그리기 공통 + 선 속성 18바이트. **구현 상태**: 구현 완료. 테스트 파일(noori.hwp)에 SHAPE_COMPONENT_LINE 레코드가 없어 실제 파일로 테스트되지 않음. **4.3.9.2.3 사각형** Tag: HWPTAG_SHAPE_COMPONENT_RECTANGLE. 표 93/94: 곡률 1바이트 + INT32 x[4], y[4]. 전체 33. **구현 상태**: 구현 완료. noori.hwp에 레코드 없음. **4.3.9.2.4 타원** 표 95/96: 전체 60. **구현 상태**: 구현 완료. noori.hwp에 레코드 없음. **4.3.9.2.5 다각형** 표 98/99. **4.3.9.2.6 호** 표 100/101: 28바이트. **4.3.9.2.7 곡선** 표 102/103: cnt, x[cnt], y[cnt], segment type[cnt-1].
