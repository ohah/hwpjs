# 4.2.8. 문단 번호

Tag ID: HWPTAG_NUMBERING

**표 38: 문단 번호**

| 자료형 | 길이(바이트) | 설명 |
|---|---|---|
| UINT | 4 | 속성(표 40 참조) |
| HWPUNIT16 | 2 | 너비 보정값 |
| HWPUNIT16 | 2 | 본문과의 거리 |
| UINT | 4 | 글자 모양 아이디 참조 |
| 전체 길이 | 8 | |

**참고**: 표 38은 각 수준(1~7)의 기본 정보를 나타냅니다. 실제 바이너리 구조에서는 표 38의 8바이트 구조와 표 39의 가변 데이터가 각 수준마다 연속적으로 배치됩니다.

**표 39: 문단 머리 정보**

| 자료형 | 길이(바이트) | 설명 |
|---|---|---|
| BYTE | 8 | 문단 머리 정보(표 40 참조) |
| stream | | 번호 형식 길이 (len) |
| WORD | 2 | 번호 형식. 불릿 문단의 경우 제어 코드 `^`와 수준 경로(예: `1,1,1,1,1,1,1`)를 포함. 7회 반복, 수준(1~7). |
| WCHAR array[len] | 2×len | 각 레벨에 해당하는 숫자 또는 문자 또는 기호를 표시 |
| UINT16 | 2 | 시작 번호 |
| UINT | 4 | 수준별 시작번호 (5.0.2.5 이상). 7회 반복, 수준(1~7) |
| WORD | 2 | 확장 번호 형식 길이 (len). 3회 반복 |
| WCHAR array[len] | 2×len | 확장 번호 형식. 수준(8~10). |

**실제 바이너리 구조**: 각 수준(1~7)마다 표 38 구조(8바이트) + 표 39 가변 데이터가 반복. 확장 번호 형식(수준 8~10)은 3회 반복.

**주의사항**: format_string이 빈 문자열이면 기본 형식, null 문자만 있으면 번호 없이 텍스트만. 확장 레벨(8~10)은 paragraph_level·line_spacing·para_style_id 등으로 결정.

**표 40: 문단 머리 정보 속성** — bit 0-1: 문단 정렬(왼쪽/가운데/오른쪽), bit 2: true/false, bit 3: 자동 내어 쓰기, bit 4: 수준별 본문과의 거리 종류(비율/값)

**표 41: 문단 번호 형식** — 0: 1,2,3 / 1: 동그라미 1,2,3 / 2: I,II,III / 3: i,ii,iii / 4: A,B,C / 5: a,b,c / 6~7: 동그라미 A,B,C / a,b,c / 8: 가,나,다 / 9: 동그라미 가,나,다 / 10: ㄱ,ㄴ,ㄷ / 11: 동그라미 ㄱ,ㄴ,ㄷ / 12: 일,이,삼 / 13: -,=,≡ / 14: 동그라미 -,=,≡
