# 부속서 I (규정) 수식 스크립트

## I.1 수식 버전

이 문서를 기준으로 수식 스크립트의 최신 버전은 6.0이다.
OWPML 본문 내에서는 수식 버전을 다음과 같이 표기한다.

```
Equation Version 60
```

## I.2 항의 구분 및 인식

수식에서 항(토큰)의 구분은 공백(Space, Tab)이나 줄바꿈으로 한다. 만약 공백을 포함한 여러 항을 하나의 항으로 인식하기 위해서는 `{`, `}`를 사용해야 한다.

- 입력: `e ^ {2x^2 + y^2}`
- 결과: e^(2x^2 + y^2)

수식에서 한 항이 9자를 넘어서면 수식 스크립트 엔진은 이를 2개의 항으로 분리하여 인식한다. 9자 이상의 항을 하나의 항으로 처리하기 위해서는 따옴표(`"`)를 항의 앞뒤에 사용해야 한다.

- 입력: `1234567890 over 5` -> 결과: 1234567890/5 (분리됨)
- 입력: `"1234567890" over 5` -> 결과: 1234567890/5 (올바름)

## I.3 수식에 쓰이는 글씨체

기본적으로 수식 편집기에서 입력하는 로마자들은 이탤릭체로 바뀌어 표현된다. 특별히 로만(영문-수식)체를 쓸 때에는 로만체 전환 명령어인 `rm`을 앞세워야 한다. 또 볼드체를 입력하기 위해서는 명령어 `bold`를 앞세운다.

| 명령어 | 설명 |
|--------|------|
| (기본) | 이탤릭체 |
| `rm` | 로만체 |
| `it` | 로만체 입력 중 이탤릭체로 변경 |
| `bold` | 볼드체 |

### I.3.1 예제

#### I.3.1.1 기본

- 입력: `Equation font test`

#### I.3.1.2 로만체 -> 이탤릭체

- 입력: `Equation rm font it test`

#### I.3.1.3 볼드체

- 입력: `rm Equation ~ bold Editor`

## I.4 White Space 표현

### I.4.1 공백 및 줄바꿈

수식 스크립트 엔진은, 공백(Space)과 줄바꿈 등은 항을 구분하는 역할로만 쓰이며 화면 표시에는 사용되지 않고 무시된다. 수식 결과에 공백을 표시하기 위해서는 `~`나 `` ` ``를 사용해야 한다. 따옴표(`"`)는 여러 개의 공백이 들어간 항을 표현할 때에도 사용될수 있다. 줄바꿈은 `#`을 사용해서 표현할 수 있다.

- 입력: `수식~편집기``예제`
- 입력: `"2차 방정식의 해법에는 인수분해에 의한"# "해법과 근의 공식에 의한 해법이 있다."`

### I.4.2 세로 칸 맞춤(탭)

세로 칸 맞춤은 맞추고자 하는 기준 글자 앞에 `&`를 입력해 주어야 한다. 즉 글자 앞에서 Tab을 누른 것과 같은 효과를 준다.

탭 없이 입력:
```
420 = 2 times 210 #
= 2 times 2 times 105 #
= 2^2 times 3 times 35 #
= 2^2 times 3 times 5 times 7
```

`&`를 사용하여 세로 정렬:
```
420 = & 2 times 210 #
= & 2 times 2 times 105 #
= & 2^2 times 3 times 35 #
= & 2^2 times 3 times 5 times 7
```

## I.5 예약어 및 명령어

예약어 및 명령어는 몇몇 예약어를 제외하고 기본적으로 대소문자를 구분하지 않는다. 대소문자를 구분하는 예약어인 경우 설명에 이를 명시한다.

### I.5.1 따옴표(`"`)

글자 길이가 긴 단어나 공백을 포함한 문장을 하나로 묶기 위해 사용되는 예약어

### I.5.2 공백 및 줄바꿈

| 예약어 | 설명 |
|--------|------|
| `~` | 보통 길이의 공백을 지칭 |
| `` ` `` | 1/4 길이의 공백을 지칭 |
| `#` | 줄바꿈을 지칭 |

### I.5.3 무한대(INF)

- 입력: `INF`
- 결과: (무한대 기호)

### I.5.4 곱셈(TIMES)

- 입력: `TIMES`
- 결과: x (곱셈 기호)

### I.5.5 분수(OVER)

분수는 기본적으로 가운데 맞춤으로 정렬한다. 오른쪽이나 왼쪽 맞춤을 쓰려면 빈칸 넣기인 `~`나 `` ` ``를 이용한다.

- 입력: `1 OVER 1 000 + {1~~~} OVER 1 000 - {~~~~1} OVER 1 000`

### I.5.6 위 아래(ATOP)

`OVER` 명령과 같지만 분수를 나타내는 가로선을 생략해 준다.

- 입력: `x ATOP y`

### I.5.7 지수(`^`, SUP)

- 입력: `x ^ y`
- 결과: x^y

### I.5.8 진수 표기 (`_`, SUB)

- 입력: `x _ 8`
- 결과: x_8

### I.5.9 제곱근(SQRT)

`SQRT`를 대신하여 `ROOT`라고 입력해도 같은 결과를 얻을 수 있다.

- 입력: `x SQRT {y + z}`
- 결과: x * sqrt(y+z)

n제곱근:
- 입력: `SQRT {3} of {y + z}`
- 결과: 3제곱근(y+z)

### I.5.10 LEFT, RIGHT

`LEFT`와 `RIGHT`가 양쪽에서 하나의 쌍을 이루도록 한다. 만약 한쪽을 생략하고 싶을 경우 생략하려는 쪽에 마침표(`.`)를 넣는다.

- 입력: `LEFT(a+b over x+y RIGHT)`
- 입력: `LEFT . a+b over x+y RIGHT }`

### I.5.11 BIGG 기호

- 입력: `LEFT { a+b OVER a-b BIGG / x+y OVER x-y RIGHT }`

### I.5.12 적분(INT, OINT)

- 입력: `INT _0^inf 1 over x dx`
- 입력: `OINT _C_1 1 over x dx`

### I.5.13 극한(lim, Lim)

본 예약어는 대소문자를 구분하므로, 반드시 대소문자를 정확하게 표기해야 한다.

- 입력: `lim _{x->0} {1 over x}` (소문자 lim)
- 입력: `Lim _{x->0} {1 over x}` (대문자 시작 Lim)

### I.5.14 합(SUM)

- 입력: `x = SUM _{i=0} ^{inf} {x sub i }`

### I.5.15 합집합(UNION), 교집합(INTER), 곱집합(PROD)

- 입력: `A UNION B` -> 결과: A U B
- 입력: `A INTER B` -> 결과: A (교집합) B
- 입력: `A PROD B` -> 결과: A (곱집합) B

### I.5.16 상호관계(REL)

두 항 간의 상호관계를 상세히 표현할 수 있게 하는 기능으로 두 항을 연결하는 화살표의 위/아래에 관계식 등의 내용을 삽입할 수 있다.

- 입력: `A REL Lrarrow abc xyz B`
- 입력: `rm PbO_2 + 2H_2 SO_4 + Pb REL exarrow 충전 방전 2PbSO_4 + 2H_2 O`

### I.5.17 BUILDREL

Relation 기능과 유사한 기능으로 화살표 아래 부분의 내용을 생략할 수 있다.

- 입력: `A BUILDREL Lrarrow { f(a) + g(a) + h(a) } B`
- 입력: `rm RCH_2 CH_2 COSCoA BUILDREL rarrow -2H~ RCH=CHCOSCoA`

### I.5.18 세로 쌓기(PILE, LPILE, RPILE)

위에 있는 문자를 기준으로 가운데(PILE), 왼쪽(LPILE), 오른쪽(RPILE) 맞춤을 선택할 수 있다.

- 입력: `PILE { t^3 # t^2 # t }`

### I.5.19 경우들(CASES)

여러 개의 행 전체를 나타내는 묶음표(`{`)는 상황에 따라 그 크기가 자동으로 확대되어 표시된다.

```
sign(x) = CASES{ 1 & x>0 #
0 & x=0 #
-1 & x=0 }
```

### I.5.20 조합(CHOOSE, BINOM)

- 입력: `n CHOOSE x`
- 입력: `BINOM n x`

### I.5.21 행렬(MATRIX, PMATRIX, BMATRIX, DMATRIX)

matrix는 줄(행) 단위로 입력하는 일반적인 방법과, matrix 다음에 col을 입력하여 칸(열) 단위로 입력하는 방법이 있다.

col을 넣어 행렬을 칸 단위로 입력할 때는 각 수식 앞에 `col`(가운데) 대신 `lcol`(왼쪽 맞춤), `rcol`(오른쪽 맞춤)을 넣어 정렬 위치를 지정할 수도 있다.

matrix 대신에 pmatrix를 쓰면 소괄호 ( ), bmatrix를 쓰면 대괄호 [ ], dmatrix를 쓰면 세로줄 |을 행렬(matrix) 양쪽에 각각 표시해 준다.

행 단위 입력:
```
matrix {
a_1 & b_1 & c_1 & d_1 #
a_2 & b_2 & c_2 & d_2 #
a_3 & b_3 & c_3 & d_3 #
a_4 & b_4 & c_4 & d_4
}
```

열 단위 입력:
```
matrix {
lcol { a_1 # abc_2 # a_3 # a_4 }
rcol { abc_1 # b_2 # b_3 # b_4 }
col { c_1 # c_2 # c_3 # c_4 }
col { d_1 # d_2 # d_3 # d_4 }
}
```

| 명령어 | 괄호 형태 |
|--------|-----------|
| `MATRIX` | 없음 |
| `PMATRIX` | 소괄호 ( ) |
| `BMATRIX` | 대괄호 [ ] |
| `DMATRIX` | 세로줄 \| \| |

### I.5.22 밑줄(UNDERLINE), 윗줄(OVERLINE)

- 입력: `UNDERLINE {x+y} + OVERLINE {y+z}`

### I.5.23 왼쪽 아래첨자(LSUB)

- 입력: `x LSUB y`
- 또는: `{}_{y} x`

### I.5.24 왼쪽 위첨자(LSUP)

- 입력: `x LSUP y`
- 또는: `{}^{y} x`

### I.5.25 최소공배수/최대공약수 함수(LADDER)

최소공배수, 최대공약수를 구할 때 사용하는 명령어로 마지막 행의 마지막 열은 인자로 인식하지 않는다.

- 입력: `LADDER {2&12&28#2&6&14#3&7&}`

### I.5.26 2진수 변환 함수(SLADDER)

10진수를 2진수로 변환할 때 사용하는 명령어로 첫 번째 행의 마지막 열과 마지막 행의 마지막 열은 인자로 인식하지 않는다.

- 입력: `SLADDER{ 2&12& # 2&6&0 # 2&3&0 # 1&1 & }`

### I.5.27 거꾸로 나눗셈(LONGDIV)

나눗셈을 표현할 때 사용하는 명령어이다.

Layout을 맞추기 위한 방법으로는 숫자만 입력해서 자동으로 Layout을 맞추거나 `~`를 이용해서 명시적으로 Layout을 맞출 수 있다. 마지막 인자에 숫자가 아닌 문자가 들어가면 자동 Layout을 하지 않는다.

```
LONGDIV {6} {422} {2532#24#13#12#12#12#0}
LONGDIV {6} {422} {2532#24#~13#~12#~~12#~~12#~~~0}
```

### I.5.28 색상(COLOR)

부분적으로 Color를 바꾸기 위해 사용하는 명령어이다. 첫 번째 인자에 `{R, G, B}` 색상값이 들어간다.

- 입력: `{COLOR {255,0,255} {3}} over {4}`

### I.5.29 글자 장식

다음의 HAT, CHECK, ARCH, TILDE 명령은 영문 3자까지만 감싸줄 수 있다.

| 입력 | 설명 | 입력 | 설명 |

| `grave A` | 음음 악센트 | `vec A` | 벡터 화살표 |
| `dot A` | 점 하나 | `dyad A` | 양방향 화살표 |
| `ddot A` | 점 두개 | `under A` | 밑줄 |
| `hat A` | 모자 (1-3자) | `arch A` | 아치 (1-3자) |
| `hat AA` | | `arch AA` | |
| `hat AAA` | | `arch AAA` | |
| `check A` | 체크 (1-3자) | `tilde A` | 물결 (1-3자) |
| `check AA` | | `tilde AA` | |
| `check AAA` | | `tilde AAA` | |
