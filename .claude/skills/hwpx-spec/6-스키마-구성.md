# 6 OWPML 스키마 구성

이 표준에서 기술하고 있는 OWPML 문서의 논리적 구조를 정의하는 XML 스키마는 5개로 구성된다.

1. **Header 스키마** — 문서 내에서 사용되는 모든 설정들을 가지고 있는 스키마이다. header에는 '구역'에 대한 정의, 금칙 문자들, 글자 설정/문단 설정과 같은 스타일 정보, 외부 참조 객체에 대한 연결 정보 등이 들어있다.

2. **Body 스키마** — 실제 문서 내용이 들어있는 스키마이다. body에는 사용자가 설정한 문서의 구분 구조대로 문서의 내용들이 저장된다.

3. **바탕쪽 스키마** — 문서에 공통으로 적용되는 바탕 쪽 설정이 들어있는 스키마이다.

4. **문서 이력 스키마** — 문서 이력 정보는 추가/수정되었던 기록을 담고 있다.

5. **버전 스키마** — 포맷에 대한 버전 스키마이다. 포맷의 버전에 대한 기록을 담고 있다.

각 스키마에 대한 주요 내용 설명은 다음 절부터 설명한다. 각 스키마에 대한 상세한 XML 스키마 정의는 첨부된 부록 XML 스키마 문서 5종을 참조한다.

## 스키마 구성 요약

| 스키마 | 네임스페이스 접두어 | 설명 | 부록 |
|--------|-------------------|------|------|
| Header XML 스키마 | `hh` | 문서 설정, 스타일, 금칙 문자, 외부 참조 등 | 부속서 C |
| Body XML 스키마 | `hb` | 문서 본문 내용 | 부속서 D |
| MasterPage XML 스키마 | `hm` | 바탕쪽 설정 | 부속서 G |
| Document History XML 스키마 | `hs` | 문서 이력 정보 | 부속서 H |
| Version XML 스키마 | `hv` | 파일 형식 버전 정보 | 부속서 B |
