# 1. 매핑 개요

## 1.1 아키텍처

```
                    ┌──── read ────┐         ┌──── write ────┐
                    │              │         │               │
HWP 5.0 (binary)  ─┤  hwp-core    ├──▶ HwpDocument ──▶ 내부 변환 ──┐
                    │  (parser)    │                              │
                    └──────────────┘                              ▼
                                                             Document  ──▶  hwp-core viewer ──▶  HTML/MD
                    ┌──── read ────┐                             ▲        (기존 코드 리팩토링)
                    │              │                              │
HWPX (XML)        ─┤  hwpx-parser ├──▶  Document  ──────────────┘
                    └──────────────┘         │
                                             ├──▶  hwpx-writer  ──▶  HWPX
                                             └──▶  hwp-writer   ──▶  HWP 5.0
```

- 뷰어 1개: hwp-core의 기존 viewer를 Document 기준으로 리팩토링
- 모델 1개: hwp-model의 Document가 유일한 공통 모델
- hwp-core 내부에 HwpDocument → Document 변환 함수 (점진 제거 예정)
- HWP write: ParaLineSeg는 기본값으로 채움 (한컴이 재조판)
- HWPX write: roundtrip fidelity 보장 목표

## 1.1.1 로드맵

| Phase | 작업 | 상태 | 비고 |
|-------|------|------|------|

| 1-5 | hwp-core viewer를 Document 기준으로 리팩토링 | ⬜ 예정 | 스냅샷 비교로 출력 동일 검증 |

## 1.2 크레이트 구조

```
crates/
├── hwp-model/          # 공통 문서 모델 (의존성 없음)
│   └── src/
│       ├── lib.rs
│       ├── document.rs     # Document, DocumentMeta, DocumentSettings
│       ├── resources.rs    # Resources, FontFaces, CharShape, ParaShape, ...
│       ├── section.rs      # Section, SectionDef, PageDef
│       ├── paragraph.rs    # Paragraph, Run, RunContent, TextContent
│       ├── control.rs      # Control enum (필드, 머리글, 각주 등)
│       ├── table.rs        # Table, TableRow, TableCell
│       ├── shape.rs        # ShapeCommon, ShapeObject enum, Picture, Line, ...
│       ├── types.rs        # HwpUnit, Color, enums, LangGroup<T>
│       └── hints.rs        # HwpHints, HwpxHints (포맷별 roundtrip 데이터)
│
├── hwpx-parser/        # HWPX → Document (read)
├── hwpx-writer/        # Document → HWPX (write, 예정)
├── hwp-writer/         # Document → HWP 5.0 (write, 예정)
│
└── hwp-core/           # HWP 5.0 parser + viewer (hwp-model 의존)
    ├── parser → HwpDocument → 내부 변환 → Document
    └── viewer: Document → HTML/MD (기존 코드 리팩토링)
```

## 1.3 설계 원칙

### 모델 기준: HWPX 트리 구조

| 결정 | 이유 |
|------|------|
| Paragraph > Run > [Text \| Control] 트리 | HWPX `<p><run><t>` 구조와 1:1 |
| Table > Row > Cell 계층 | HWPX `<tr><tc>` 구조와 1:1 |
| ShapeCommon 공통 속성 | HWPX AbstractShapeObjectType 그대로 |
| LangGroup<T> 제네릭 | 7개 언어별 속성을 타입 안전하게 |
| HwpValue { value, unit } | HWPX 단위 정보(HWPUNIT/CHAR) 보존 |
| SubList 재귀 컨테이너 | 캡션, 셀, 머리글 등 동일 패턴 |

### 포맷별 필드 분류 원칙

```
"이 데이터가 없으면 문서의 의미가 달라지는가?"
  → Yes: 공통 모델 (Option 가능)
  → No:  hints

"양쪽 다 표현할 수 있는가?"
  → Yes: 공통 모델 (필수)
  → No:  있는 쪽에서 읽고, 쓸 수 있는 writer가 처리
```

### HWP 5.0 파서의 추가 작업

HWP 5.0의 평면 레코드를 트리로 조립하는 변환이 필요:

| HWP 평면 구조 | 공통 모델 트리 구조 | 변환 |
|-------------|------------------|------|
| ParaText + ParaCharShape | Paragraph > Run[] | 위치 경계에서 텍스트 분할하여 run 생성 |
| Table.cells (Vec) | Table > Row[] > Cell[] | row/col 좌표 기준 그룹핑 |
| CtrlHeader + ObjectCommon + ShapeComponent | ShapeObject { common, ... } | 분산 데이터 합치기 |
| 비트마스크 (UINT32) | 개별 bool/enum 필드 | 비트 추출 |

## 1.4 매핑 결과 요약

### 대응 현황

| 분류 | 비율 | 설명 |
|------|------|------|
| 100% 직접 매핑 | ~90% | 필드명/타입만 다르고 의미 동일 |
| 표현만 다름 (변환 필요) | ~7% | 비트마스크 ↔ XML 속성, 평면 ↔ 트리 |
| HWP only → hints | ~2% | ParaLineSeg, ControlMask, 캐럿 위치 |
| HWPX only → Option | ~1% | MetaTag, isReverseHV, intervalDirty |

### HWP only (hints)

| 항목 | 용도 | 모델 위치 |
|------|------|----------|
| ParaLineSeg | 레이아웃 캐시 | HwpParagraphHints |
| ControlMask | 바이너리 파싱 편의 | HwpParagraphHints |
| 캐럿 위치 (list_id, para_id, char_pos) | 에디터 상태 | HwpDocumentHints |
| Language ID (구역 정의) | 레거시 | HwpSectionHints |
| 문단 꼬리 모양 (bit 30) | 레거시 | HwpParagraphHints |
| FileHeader 원본 256바이트 | roundtrip | HwpDocumentHints |

### HWPX only (Option)

| 항목 | 용도 | 모델 위치 |
|------|------|----------|
| MetaTag (JSON) | 문단/개체 메타데이터 | Paragraph.meta_tag, ShapeCommon.meta_tag |
| isReverseHV (line) | 선 방향 교정 | LineObject.is_reverse_hv |
| intervalDirty (ellipse) | 간격 재계산 플래그 | EllipseObject.interval_dirty |
| dropcapStyle | 드롭캡 스타일 | ShapeCommon.dropcap_style |
| Dublin Core 메타데이터 | 표준 문서 메타 | DocumentMeta |
| memoShapeIDRef (섹션) | 메모 참조 | SectionDef.memo_shape_id |
| masterPageCnt (섹션) | 바탕쪽 개수 | SectionDef.master_page_cnt |

## 1.5 단위

양쪽 동일:

```
1 HWPUNIT = 1/7200 inch
1 pt = 100 HWPUNIT (charPr@height 기준)
1 mm = 283.456 HWPUNIT
1 cm = 2834.56 HWPUNIT
1 inch = 7200 HWPUNIT
```

색상: HWP는 COLORREF (u32, 0x00BBGGRR), HWPX는 #RRGGBB 문자열 → 공통 모델은 u32 (0x00RRGGBB)
