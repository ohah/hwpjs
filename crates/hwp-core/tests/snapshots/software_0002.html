SW 내부 설계
주요 기능 흐름
1. 교사 로그인 및 수업개설
- 수업 생성 서버에서 SSE채널 초기화
- 각 투자 라운드 별 기사/기업데이터 세팅
2. 학생 로그인 및 투자 참여
- 1차 투자 진행 (기업 9곳 중 선택, 500만원 한도 내 투자 결정)
- 기업별 재무제표, 주요 기사 확인
- 교사가 기사 공개 시 SSE를 통해 실시간 브로드 캐스트
3. 1차 투자 마감 -> 신문기사 공개
- 새로운 기사 공개 시 학생들은 정보 분석 후 2차 투자 진행
- 매수/매도 요청 서버에서 SSE를 통해 반영 결과 실시간 전송
4. 2차 투자 및 결과 조회
- 학생별 투자 종목, 보유 수량, 평가 금액, 수익률 등 확인 가능
- 투자 후 수정 불가 (1포인트 내 . 회 매수/매도 제한)
- 잔액 없을 시 . ㅐ수 불가, 매도 후 재투자 가능
5. 투자 종료 및 결과 발표
- 투자 종료 서버가 최종 수익률 계산 후 랭킹 발표
- 교사 페이지에서 전체 결과를 시각화 (그래프, 수익률 차트 등)
시스템 구성도
- TB_ADMIN: 관리자(교사) 계정 관리
- TB_CLASS: 수업 세션 및 라운드 정보
- TB_ART: 뉴스 기사 테이블
- TB_ITEM: 투자 가능한 종목 정보 (기업명, 재무 정보, 가격 등)
- TB_CLASS_ITEM: 클래스별 투자 종목 연결
- TB_ITEM_ORDER: 매수/매도 내역(거래 시간 금액 수량등)
- TB_CLASS_ITEM: 학생별 참여 그룹 정보
- TB_CLASS_ART: 각 수업에서 공개된 기사 기록
                                                                                                                                      - HOLD_ITEM: 현재 보유 주식 현황, 평가 금액, 수익률 저장
