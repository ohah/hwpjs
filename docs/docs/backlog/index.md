# 백로그

이 디렉토리는 프로젝트의 백로그와 구체적인 작업 항목을 관리합니다.

## 디렉토리 구조

```
backlog/
├── _meta.json           # Rspress 메타데이터
├── index.md             # 이 파일 (백로그 개요)
└── 10_current_tasks.md  # 현재 진행 중인 작업
```

## 파일별 용도

### `index.md` (이 파일)
- 백로그 디렉토리 구조 설명
- 백로그 관리 방법
- 로드맵과의 관계

### `10_current_tasks.md`
- 현재 진행 중인 작업 항목
- 우선순위가 높은 태스크
- 진행 상황 추적

## 백로그 관리 방법

1. **작업 추가**: 새로운 작업이 생기면 `10_current_tasks.md`에 추가
2. **상태 표시**: 각 작업 항목에 진행 상태 표시 (진행 중, 완료, 대기 등)
3. **우선순위**: 중요도에 따라 작업 순서 결정
4. **완료 처리**: 완료된 작업은 체크박스로 표시하거나 별도 섹션으로 이동

## 로드맵과의 관계

- **로드맵**: 전략적 계획 및 장기 목표 (상위 개념)
- **백로그**: 로드맵을 달성하기 위한 구체적인 작업 항목 (하위 개념)

백로그의 작업들은 로드맵의 목표를 달성하기 위해 수행되는 구체적인 태스크입니다.
